# What Is Version Control

## Purpose
- Version control records changes to files over time.
- It enables collaboration, history, branching, review, and rollback.
- Git is the dominant distributed version control system.

## Key concepts
- Repository stores project history.
- Commit is a snapshot with message and author.
- Branch isolates a line of work.
- Merge combines changes.
- Tag marks a release point.

## Exam relevance
- CI/CD starts from version-controlled changes.
- Infrastructure as Code also belongs in version control.
- Without version control, auditability and repeatable deployment are weak.
