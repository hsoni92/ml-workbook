# Week 12: DevOps and CI/CD Foundation

## DevOps idea
- DevOps combines culture, automation, measurement, and shared ownership to deliver software reliably.
- The point is not a tool; the point is shorter feedback loops and safer change.
- Cloud platforms make DevOps practical through APIs, managed environments, and automated deployment targets.

## CI/CD flow
- Source change enters version control.
- CI builds and tests automatically.
- Artifacts are versioned and stored.
- CD promotes artifacts through environments with verification and rollback.

## Exam focus
- CI is integration validation; CD is release/delivery automation.
- Deployment strategy choice depends on risk, downtime tolerance, and rollback need.
- Manual testing-only pipelines are not real CI/CD.
