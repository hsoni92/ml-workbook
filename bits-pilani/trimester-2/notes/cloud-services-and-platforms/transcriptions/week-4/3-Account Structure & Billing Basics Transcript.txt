In this lesson, we'll focus on three very important AWS fundamentals, which is AWS

account is the basic container where everything in AWS lives.

to an AWS account. AWS uses accounts as strong boundaries, whether it is for
security, for billing, and most importantly, for isolation. That's why organisations often
use multiple accounts, such as one for development, one for testing, and one for the
production, which is actually accessible to the customers.
This separation helps limit mistakes, control the access, and track the costs clearly.
Now, when an AWS account is created, AWS automatically creates one special user,
which we refer to as a root user. Root user has full and unrestricted access to the
entire account.
It can do things that no other user can, such as closing the account, changing the
billing ownership, and many more. Because of this power, the root user should never
be used for daily work. Using it regularly increases risk, because if those credentials
are compromised, the entire account is exposed.
So, the best practise is to secure the root user with a multi-factor authentication, which
we are going to explore in the upcoming videos, and use it only for critical account -
level tasks. Let's now talk briefly about billing. AWS follows a pay -as-you-go model.

One important thing for beginners to understand is that running a resource always
costs money.
