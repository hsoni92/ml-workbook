IP.

launched. If the subnet setting will allow this, it will automatically be launched and
assigned.
This IP allows the instance to communicate with the internet. But public IPs are usually
temporary. If you stop and start the instance, the public IP will change.

lived servers where the IP stability doesn't matter much. Now let's move to Elastic IPs.
An Elastic IP is a static public IP that you can explicitly allow to your AWS account.
This IP remains the same even if the instance stops or restarts. You can also detach
it from one instance and attach it to another within seconds.
This makes Elastic IP extremely useful for production issues or any kind of failover
