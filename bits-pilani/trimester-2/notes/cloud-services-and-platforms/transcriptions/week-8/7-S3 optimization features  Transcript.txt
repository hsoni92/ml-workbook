S3 optimization features transcript

In this topic, we are going to talk about something that quietly saves or wastes a lot
of money in S3. Versioning, lifecycle rules and cost optimisation. Let's start with
versioning.

S3 versioning means that when a file is updated or deleted, the old version is not
immediately lost. Instead, S3 keeps multiple versions of the same object. This is
extremely useful because accidental deletions and overwrites are very common in
real systems.

Although it's not encouraged, but it is common. Think of versioning like a rewind
button. If something goes wrong, you can always go back to the earlier version.

When versioning is enabled, uploading the same file name creates a new version.
Deleting a file doesn't remove it. S3 simply adds a delete marker.

So older versions are still there and can be restored. However, there is an important
trade-off here. Every version consumes storage and storage costs money.

So versioning should always be paired with lifecycle rules. Now let's talk about
lifecycle rules. These rules automate what happens to data over time.

Instead of manually cleaning old files or moving them to a cheaper storage, you
define rules once and S3 will apply them automatically.

For example, you might keep application logs in standard storage for 30 days, move
them to infrequent access after that, then archive them in the glacier and finally
delete them after a year. This is important because most data is only valuable when
it's new.

As it gets older, access drops but storage costs will still continue unless you have
managed it. So lifecycle rules help you keep storage clean and predictable. Now
coming to the cost optimisation.

S3 itself is cost effective but only when you use it correctly. The biggest cost mistake
happens when teams keep old versions forever, store the cold data in expensive
storage classes like standard or forget lifecycle rules entirely.So good S3 design
means matching storage type to access pattern.

Frequently accessed data stays in standard, occasionally accessed data moves to
infrequent access, rarely accessed data goes to glacier or deep archive.

In short, versioning protects your data, lifecycle rules manage its lifespan and
together they keep your S3 costs under the control.

This combination is what separates S3 usage from a beginner's point of view to a
production-grade cloud design.
