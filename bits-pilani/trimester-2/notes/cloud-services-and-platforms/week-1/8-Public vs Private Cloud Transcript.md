# Public Cloud vs Private Cloud

## Objective
Understand the difference between public and private cloud deployment models, and when each is selected.

## Public Cloud

### Definition
Cloud services delivered over the internet on shared provider infrastructure (multi-tenant platform).

### Characteristics
- Pay-as-you-go pricing
- No hardware ownership required
- Fast provisioning
- High elasticity and geographic scale

### Typical Providers
- AWS
- Microsoft Azure
- Google Cloud Platform

### Strengths
- Low upfront cost
- Fast scaling
- Rapid global rollout
- Reduced maintenance burden

### Typical Users
- Startups
- Digital-native companies
- Businesses needing flexibility and fast growth

## Private Cloud

### Definition
Dedicated cloud environment for a single organization.

### Deployment Options
- Hosted in organization data center
- Managed by third-party provider
- Built using private cloud technologies (for example, VMware/OpenStack ecosystems)

### Strengths
- High control
- Greater customization
- Support for strict compliance and regulated workloads

### Typical Users
- Banks and financial institutions
- Government organizations
- Highly regulated enterprises

## Key Comparison
| Dimension | Public Cloud | Private Cloud |
|---|---|---|
| Infrastructure | Shared | Dedicated |
| Upfront Cost | Low | High |
| Control | Lower | Higher |
| Scalability | Very high | Limited by owned capacity unless expanded |
| Compliance Handling | Strong, but with shared model constraints | Easier for strict data residency/control requirements |

## Decision Guidance
- Choose **public cloud** for cost efficiency, speed, and scalability.
- Choose **private cloud** for strict control, governance, and sensitive data handling.

## Exam-Focused Questions
- Explain multi-tenancy without implying data mixing.
- Compare public vs private on cost, control, and compliance.
- Give one practical industry example for each model.

## Quick Revision
- Public cloud optimizes speed and economics.
- Private cloud optimizes control and compliance.
- Both are valid; business context determines choice.
