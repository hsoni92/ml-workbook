# Cloud Evolution

## Learning Objectives
- Trace cloud computing from early shared computing to modern hyperscale platforms.
- Understand why each historical stage mattered.
- Connect history with current business and technology adoption patterns.

## Timeline of Evolution

### 1) 1960s: Time-Sharing Era
- Computers were very expensive and scarce.
- Multiple users shared one powerful machine in allocated time slots.
- Key idea introduced: **shared access to computing resources**.
- This is the conceptual seed of modern cloud economics.

### 2) 1990s: Virtualization Breakthrough
- Virtualization made it possible to run multiple virtual machines on one physical server.
- Organizations improved hardware utilization and reduced cost per workload.
- Benefits:
  - Better resource efficiency
  - Faster provisioning than pure physical infrastructure
  - More flexible workload isolation

### 3) 2006: Public Cloud Commercialization (AWS Launch)
- Major shift: servers could be rented over the internet on demand.
- Provisioning changed from weeks/months to minutes/seconds.
- Startups avoided large upfront capital expenditure.
- Enterprises accelerated experimentation and product development.

### 4) 2010 Onwards: Multi-Provider Growth
- Azure and Google Cloud expanded market maturity.
- Cloud became mainstream for startups and large enterprises.
- Competition increased service quality, geographic reach, and specialized capabilities.

### 5) Cloud-First Modern Era
- Cloud now powers streaming, mobility, storage, AI training, and many critical industries.
- Cloud is no longer optional infrastructure; it is often the default platform.

## Conceptual Analogy
Using cloud is like using a ride service instead of owning a car:
- Ownership model: high upfront cost + maintenance burden
- Usage model: pay only when needed

Cloud follows the same principle:
- No mandatory hardware ownership
- Usage-based pricing
- Operational burden shifted to provider

## Business Relevance
- Faster go-to-market for digital products
- Reduced initial investment risk
- Elastic scale for unpredictable demand
- Better innovation velocity for both startups and enterprises

## Exam-Focused Key Points
- Time-sharing = first resource-sharing model in computing history.
- Virtualization = technical enabler for efficient cloud infrastructure.
- AWS 2006 = major commercialization inflection point.
- Modern cloud adoption is driven by speed, flexibility, and economics.

## Quick Revision
- 1960s: shared computer time -> 1990s: virtualization -> 2006: on-demand cloud -> 2010+: hyperscale adoption.
- Cloud evolution is a shift from **ownership** to **consumption**.
