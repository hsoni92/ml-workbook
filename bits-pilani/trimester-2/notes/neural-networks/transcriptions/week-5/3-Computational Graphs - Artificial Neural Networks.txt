Why computational graphs make backpropagation efficient and systematic? A computational graph is a graphical representation of a mathematical expression. In this graph, nodes represent variables or intermediate values. Edges represent mathematical operations that transform one value into another.
The graph shows us exactly how a final output is constructed step by step from the input. Let us look at a very simple example. We define u equals 3x and then y equals u square.
As a graph, this becomes a simple chain. The input node is x. The first operation computes u equals 3x. The second operation computes y equals u square. So, even this small expression already has a clear graph-like structure.

This shows us a key idea. A global gradient is the product of local gradients along a path in the graph. The same computational graph is used both for the forward pass and the backward pass. Any neural network computation can be represented as a computational graph. Each edge in the graph has a local gradient. Global gradients are computed by multiplying local gradients along the paths using the chain rule. Backpropagation is simply the systematic backward traversal of this computational graph to compute all parameter gradients efficiently.
