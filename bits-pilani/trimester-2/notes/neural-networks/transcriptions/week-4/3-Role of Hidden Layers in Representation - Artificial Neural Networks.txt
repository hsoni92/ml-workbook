Let us begin with a simple question. Why do we need hidden layers at all? A single layer can only learn linear transformations of the input. No matter how many neurons we use, without hidden layers, we are still limited to linear decision boundaries.
However, most real-world relationships are highly non-linear. Think of images, speech signals, medical data, or financial patterns. These cannot be captured using only linear mappings.
Hidden layers allow the network to go beyond raw inputs. Their core role is to transform and re-encode the data into new internal representations that make complex patterns easier to separate.
When we talk about representation, we simply mean how the data is encoded inside the network. The raw input, such as pixels of an image or values in a table, is the first representation. Each hidden layer produces a new transformed version of the same data.
If we go back to the housing price prediction problem, we had inputs like number of bedrooms and size of the house. These two together could determine the spaciousness of the house.
Now, this spaciousness is not present in the input features, but it is part of the hidden layer, which is an abstract information being learned by the model.
This is a key idea in deep learning. Learning is not only about learning a classifier. It is also about learning better internal representations of the data.
Now, let us zoom into the level of a single hidden neuron. Mathematically, a hidden neuron computes A equals function of W transpose X plus B.
Conceptually, this means the neuron responds strongly when a specific pattern appears in the input. It is therefore useful to think of a hidden neuron as a feature detector.
Different hidden neurons learn to respond to different patterns in the same input. Together, they form a bank of feature detectors.
Let us now connect this to the idea of raw versus derived features. Raw features are direct measurements. For example, pixel intensities in an image. By themselves, these raw features are often not very informative.
Hidden layers transform these raw features into derived features. These derived features can capture combinations of inputs, interactions between inputs, structural patterns in the data. Each new layer works not on the original input, but on features created by the previous layer.
When we stack multiple hidden layers, something very important happens. The network begins to learn hierarchies of representations. Early layers usually learn simple patterns such as edges or basic numeric trends. Deeper layers then learn combinations of those patterns such as shapes, objects or more abstract concepts.
If you look at the network here, the idea here is to classify an image into the categories dog, cat or bird. In the input layer, we have the raw pixels. In the first hidden layer, the network tries to identify the edges. In the second layer, it is learning the corners and contours. As we go deeper into the network, it is finally combining all the patterns learnt in initial layers to identify the shapes and objects.
This process is called hierarchical representation learning. It is one of the central reasons why deep networks are more powerful than the shallow ones. However, this increased representation power does not come for free. More hidden neurons mean more feature detectors. More hidden layers mean more levels of abstraction. Together, these greatly increased representation.
These greatly increased the set of functions that the network can represent. But they also increase the number of parameters, the computational cost and the risk of overfitting. So, there is always a trade-off between representation power and model complexity. Later, when we study training and regularization, we will see how this trade-off is managed in practice.
