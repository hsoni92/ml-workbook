Let's compare batch and micro batch
side by side. Latency. Batch has a
higher latency, hours, or a full day.
Micro batch brings that down to minutes.
Complexity, batch is simpler, micro
batch is more complex because you have
more frequent runs and more things to
monitor. Cost batch is fewer big jobs,
micro batch is many smaller jobs which
can add overhead. Use case, batch is
perfect for offline retraining and
score all jobs. Micro batch is ideal
for near real time features and metrics.
A good rule of thumb, start with batch
and. Only move to micro batch when you
clearly need fresher data.
