Let's first answer the basic question.
What does an optimised run time
actually do? Imagine you have a model
graph exported to a standard format
like ONNX, which stands for Open Neural
Network Exchange. The runtime will
optimise the graph, for example, by
fusing a convolution, batch
normalisation, and RELU into the single
operation. Choose the fastest kernel
implementation it can find for each
operation on that target hardware,
whether that's a central processing
unit or CPU or graphics processing unit
or GPU.
Manage memory, reusing buffers instead
of constantly allocating and freeing
them.
And make use of any hardware
accelerator, such as vector
instructions, GPUs. Neural processing
units, or NPUs, or tensor processing
units, TPUs. The goal is simple.
Keep the model's output the same, but
reduce latency, improve throughput, and
often reduce memory usage.
