The model parameters and configurations

machine learning pipeline.

First, let's talk about data and model.
When you ship a new model, you will
implicitly ship shipping a particular
training data set version, a particular
label, definition, and labelling
process. A snapshot of the data
distribution at that time. If the data
changes, say features disappears,
distribution dev shifts.
Our labels are defined differently, the

schema as part of the release. Run data
quality checks. They are field present
are range scenes.
Is there unexpected drift compared to
previous runs? In other words, every

new code, it's code plus data.
The second big difference is that
models themselves are artefacts.
And machine learning pipeline doesn't

serialised models.
Evaluation metrics, AUC accuracy, loss

version these models, attach their
metrics to each version, and use those
metrics to decide whether a model is
eligible for deployment. So machine
learning pipelines are not just
Did the test pass?
There also, does this model meet or
exceed the performance criteria
compared to the baseline?
Another key difference in machine
